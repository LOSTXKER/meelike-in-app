#### API Integration Code

```typescript
// ===== PAYOUT SERVICE =====

interface PayoutRequest {
  id: string;
  workerId: string;
  amount: number;           // ยอดถอน (หลังหักค่าธรรมเนียม)
  feeAmount: number;        // ค่าธรรมเนียม
  method: 'promptpay' | 'bank_transfer';
  
  // PromptPay
  promptpayNumber?: string;
  
  // Bank Transfer
  bankCode?: string;        // 'KBANK', 'SCB', 'BBL'
  bankAccountNumber?: string;
  bankAccountName?: string;
  
  status: 'pending' | 'approved' | 'processing' | 'completed' | 'failed';
  
  // Timestamps
  requestedAt: string;
  approvedAt?: string;
  processedAt?: string;
  completedAt?: string;
  
  // Result
  bankRefNumber?: string;
  errorMessage?: string;
}

// SCB Open API Example
interface SCBTransferRequest {
  transactionId: string;
  transactionDateTime: string;
  senderBankCode: string;
  senderAccountNo: string;
  receiverBankCode: string;
  receiverAccountNo: string;   // หรือ PromptPay number
  transferAmount: string;
  paymentRef: string;
}

// Transfer Function
async function processPayoutViaSCB(payout: PayoutRequest): Promise<boolean> {
  const scbApi = new SCBOpenAPI({
    apiKey: process.env.SCB_API_KEY,
    apiSecret: process.env.SCB_API_SECRET
  });
  
  try {
    const result = await scbApi.transfer({
      transactionId: payout.id,
      transactionDateTime: new Date().toISOString(),
      senderBankCode: '014',      // SCB
      senderAccountNo: process.env.SCB_ACCOUNT_NO,
      receiverBankCode: payout.method === 'promptpay' ? '' : payout.bankCode,
      receiverAccountNo: payout.method === 'promptpay' 
        ? payout.promptpayNumber 
        : payout.bankAccountNumber,
      transferAmount: payout.amount.toFixed(2),
      paymentRef: `PAYOUT-${payout.id}`
    });
    
    // Update payout status
    await updatePayout(payout.id, {
      status: 'completed',
      bankRefNumber: result.transactionRef,
      completedAt: new Date().toISOString()
    });
    
    // Notify Worker
    await notifyWorker(payout.workerId, {
      type: 'payout_success',
      amount: payout.amount,
      refNumber: result.transactionRef
    });
    
    return true;
    
  } catch (error) {
    await updatePayout(payout.id, {
      status: 'failed',
      errorMessage: error.message
    });
    
    // Alert Admin
    await alertAdmin(`Payout failed: ${payout.id} - ${error.message}`);
    
    return false;
  }
}
```

### 🎯 QC Model: Seller-Managed (Platform ไม่ตรวจงาน)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│  💡 หลักการ: Seller ตรวจงานเอง, Platform จัดการระบบ                         │
│                                                                              │
│  ═══════════════════════════════════════════════════════════════════════    │
│                                                                              │
│  ❌ แบบเดิม (Platform ตรวจ):                                                 │
│  Worker ส่งงาน → Platform ตรวจ → Seller ได้งาน                              │
│                       ↑                                                      │
│                  (ไม่ Scale!)                                               │
│                                                                              │
│  ✅ แบบใหม่ (Seller ตรวจ):                                                  │
│  Worker ส่งงาน → Seller ตรวจ + Approve → Worker ได้เงิน                     │
│                       ↑                                                      │
│              (Seller รู้ดีที่สุด!)                                          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

| Approve/Reject งาน | ❌ | ✅ |
| กำหนดเงื่อนไขงาน | ❌ | ✅ |
| จัดการเงิน (Escrow) | ✅ | ❌ |
| Notification | ✅ | ❌ |
| Dispute Resolution | ✅ (เฉพาะกรณี) | ❌ |
| Rating System | ✅ | ❌ |
| รับผิดชอบคุณภาพงาน | ❌ | ✅ |

**กระบวนการ: Manual + AI (ไม่เก็บค่า Verify)**

```

│  │ ลงทะเบียน│ ──>│ Screenshot│ ──>│ ตรวจสอบ │ ──>│ รอเปิดใช้│                   │
│  │ บัญชี   │    │ หลักฐาน │    │         │    │ งาน    │                   │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘                   │
│                                      │                                      │
│                                      ▼                                      │
│                               ┌─────────┐                                   │
│                               │ 👤 Manual│  (กรณี AI ไม่แน่ใจ)              │

│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### ขั้นตอนการยืนยัน:

1. **Worker ลงทะเบียนบัญชี Social**
   - เลือก Platform (Facebook, Instagram, TikTok, etc.)
   - กรอก Username / Profile URL
   - อัพโหลด Screenshot หน้าโปรไฟล์

2. **AI ตรวจสอบอัตโนมัติ** (OpenAI Vision)
   - ✅ มีรูปโปรไฟล์
   - ✅ จำนวน Followers ตามเกณฑ์
   - ✅ ไม่ใช่หน้า Login / Error
   - ✅ Username ตรงกับที่กรอก

3. **ผลลัพธ์**
   - ✅ **ผ่าน** → บัญชีพร้อมใช้งาน
   - ⚠️ **ไม่แน่ใจ** → ส่ง Admin ตรวจ Manual
   - ❌ **ไม่ผ่าน** → แจ้งเหตุผล + ให้ลองใหม่

#### สิ่งที่ AI ตรวจสอบ:

| รายการ | ตัวอย่าง |
|--------|---------|
| 📷 รูปโปรไฟล์ | ต้องมีรูป (ไม่ใช่รูป default) |
| 👤 ชื่อบัญชี | ตรงกับที่กรอก |
| 👥 Followers | ≥ 50 (ขั้นต่ำของระบบ) |
| 📅 ความเก่า | ดูจาก UI (ถ้าเห็น) |
| 🚫 ปลอม | ตรวจ Screenshot ปลอม |

#### ต้นทุน AI (OpenAI Vision):

```
ราคา: ~$0.003/รูป (~฿0.11/รูป)

ตัวอย่าง:
- 100 บัญชี/เดือน = ฿11
- 1,000 บัญชี/เดือน = ฿110
- 10,000 บัญชี/เดือน = ฿1,100
```

**หมายเหตุ:** ไม่เก็บค่า Verify จาก Worker (เป็นบริการฟรี)

**Worker:**
- ⏰ ส่งงานตรงเวลา: +0.1
- 📷 หลักฐานชัด: +0.1

## 📝 Notes

### ข้อดีของโครงสร้างนี้

1. **MeeLike ไม่ต้องแก้ไข** - อยู่เหมือนเดิม แค่เพิ่ม Banner
2. **MeeLike Seller focus ชัด** - Platform สำหรับ Seller/Worker/Buyer
3. **Scalable** - แยก scale ได้
4. **Revenue หลายทาง** - Sub + Fee + Margin
5. **User Experience ดี** - ทุกอย่างอยู่ที่เดียว

### ความเสี่ยง

1. **Adoption** - ต้อง Marketing ดี
2. **Trust** - ระบบ Escrow ต้องน่าเชื่อถือ
3. **Quality Control** - Worker ต้องทำงานดี
4. **Competition** - อาจมีคู่แข่ง

### Success Metrics

- จำนวน Seller ที่สมัคร
- จำนวน Worker ที่ active
- ยอด Order / เดือน
- ยอดขายบัญชี / เดือน
- Revenue / เดือน
- User Retention
